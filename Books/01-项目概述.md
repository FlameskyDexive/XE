## 1.1 什么是 Prowl

Prowl 是一款 **MIT 开源**、**纯 C# / 最新 .NET** 实现的游戏引擎，目标是：

- 提供接近 Unity 的 API 与编辑器体验，降低迁移成本；
- 遵循 KISS，保持代码体量可控、可定制；
- **运行时（Prowl.Runtime）可完全脱离编辑器独立运行**——可直接引用 Runtime 库发布游戏。

当前版本为 **1.0-preview**。此前版本的项目 **不兼容** 且 **无迁移路径**；1.0 正式版发布前仍可能发生破坏性变更。

---

## 1.2 设计哲学

| 原则 | 体现 |
|------|------|
| Unity 亲和 | GameObject / MonoBehaviour、Scene、Prefab、Asset GUID、Input Action 等概念与 API 形态 |
| 运行时独立 | Runtime 无 Editor 依赖；Samples 全部仅引用 Runtime |
| 可扩展 | 自定义 Importer、自定义 RenderPipeline、自定义编辑器、Assembly Definition、Plugin |
| 显式控制 | 渲染走 CommandBuffer + 专用渲染线程；序列化走 Prowl.Echo；资源走 GUID |
| 跨平台 | 编辑器与导出构建均支持 Windows / Linux / macOS |

## 1.3 仓库与工程结构

```
Prowl/
├── Prowl.sln
├── Prowl.Runtime/          # 核心运行时库 (net10.0)
├── Prowl.Editor/           # 编辑器可执行程序
├── Prowl.Runtime.Test/     # Runtime 单元测试 (xUnit)
├── Prowl.Editor.Test/      # Editor 单元测试
├── Samples/                # 仅依赖 Runtime 的示例工程
├── Libraries/              # 各平台原生库 (MiniAudio 等)
├── BRDFGen/                # BRDF LUT 生成工具
├── Books/                  # 本技术文档目录
├── README.md
└── CONTRIBUTING.md
```

### 解决方案项目一览

| 项目 | 类型 | 输出 | 依赖 |
|------|------|------|------|
| **Prowl.Runtime** | 类库 | `Build/Runtime/{Config}/` | Silk.NET、Jitter2、Magick.NET、Prowl.* 系列 |
| **Prowl.Editor** | 可执行 | `Build/Editor/{Config}/` | → Runtime + Origami / Photonic / Rosetta |
| **Samples/\*** | 可执行 | 各示例独立 | → **仅** Runtime |
| **\*.Test** | 测试 | — | xUnit + NSubstitute |

## 1.4 主要功能地图

### 通用能力

- 跨平台编辑器与玩家构建（Win / Linux / Mac）
- C# 脚本（.NET 10）+ 热重载
- GameObject / MonoBehaviour 组件架构
- 场景、标签与层、Prefab（含嵌套与 Override）
- 基于 GUID 的资源数据库与 `.meta` 文件
- 独立 Immediate Mode UI（Paper）与编辑器组件库（Origami）
- 构建系统（`.prowlpak` 打包、按依赖导出）
- 450+ 单元测试

### 图形

- OpenGL 后端（Silk.NET）+ **专用渲染线程**
- 可扩展 RenderPipeline；默认 Forward + Thin G-Buffer Prepass
- PBR（Metallic）、HDR、级联阴影 / 点光立方体阴影 / 阴影图集
- 后处理：Tonemap、Bloom、FXAA / TAA / SMAA、GTAO、SSR、DOF、体积雾、电影感效果等
- 蒙皮网格、BlendShape、GPU Instancing、Terrain、粒子

### 物理 / 音频 / 输入

- **Jitter Physics 2**：多种碰撞体、约束/关节、角色控制器、车轮、射线/形状查询
- **MiniAudio**：3D 空间音频、多普勒、效果链
- **Input Action**：绑定、Composite、处理器、阶段机

## 1.6 构建与运行前提

1. 安装 [.NET 10 SDK](https://dotnet.microsoft.com/download/dotnet/10.0)
2. 克隆仓库，用 VS 17.8+ / VS Code / Rider 打开 `Prowl.sln`
3. 构建 `Prowl.Editor` 即可启动编辑器；构建任意 `Samples/*` 可运行无编辑器 demo

更细的运行说明见仓库根目录 `Run.md` 与 `README.md`。
