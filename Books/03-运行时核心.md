# 03 · 运行时核心

---

## 3.1 Application — 全局模式标志

**文件:** `Prowl.Runtime/Application.cs`

`Application` 是纯静态类，不持有实例生命周期，只表达「当前进程/时刻的运行语义」：

| 成员 | 含义 |
|------|------|
| `IsPlaying` | 是否处于游戏运行态（玩家启动后恒 true；编辑器由 Play Mode 驱动） |
| `IsEditor` | 是否在编辑器进程 |
| `IsHeadless` | 无窗口 / 无 GPU / 无音频（CI、服务器） |
| `IsPaused` / `StepRequested` | 暂停与单帧步进 |
| `ShouldRunGameplay` | `IsPlaying && (!IsPaused \|\| StepRequested)` |
| `IsGameplayExecuting` | 正在跑 FixedUpdate/玩法 tick 时为 true（供编辑器过滤输入） |
| `DataPath` | 玩家资源根路径 |

脚本与系统应依赖这些标志做分支，而不是硬编码 `#if`。

## 3.2 Game — 主循环宿主

**文件:** `Prowl.Runtime/Game.cs`

抽象类，模板方法模式。派生类（编辑器的 `EditorApplication`、玩家的 `DesktopPlayer`、各 Sample）实现业务钩子。

### 入口

```csharp
public void Run(string title, int width, int height);   // 窗口循环
public void RunHeadless(HeadlessRunOptions? options);   // 无头循环
public void Quit();
public void RequestHeadlessQuit();
```

### 窗口路径关键步骤

1. 强制 `CultureInfo.InvariantCulture`（材质/着色器数值解析一致性）
2. `Application.IsPlaying = true`（编辑器稍后覆盖）
3. `Window.InitWindow` + 事件绑定
4. `Load`：`AudioContext.Initialize` → Paper UI → `BuiltInAssets.Initialize` → `Initialize()`
5. `Update` → `SimulationStep`
6. `Render` → 场景渲染 + Paper
7. `Closing` → `Closing()`、`Scene.Unload()`、音频拆解

### 可覆盖钩子

| 虚方法 | 默认行为 |
|--------|----------|
| `Initialize` / `Closing` | 空 |
| `BeginUpdate` / `EndUpdate` | 空 |
| `BeginRender` / `EndRender` | 空 |
| `BeginGui` / `EndGui` | 空 |
| `OnUpdate(Scene?)` | `scene?.Update()`（+ 可选 Gizmo） |
| `OnRender(Scene?)` | `scene?.Render()` |
| `OnGui(Scene?, Paper)` | `scene?.OnGui(paper)` |
| `Resize` | 空 |
| `ResetFixedTimeAccumulator()` | 进入 Play 时重置固定步长累加器 |

编辑器会 **关闭** 默认的 `OnRender`/`OnGui` 路径，改由 `SceneViewPanel` / `GameViewPanel` 驱动，详见 [12-编辑器架构](12-编辑器架构.md)。

## 3.4 GameObject

**文件:** `Prowl.Runtime/GameObject/GameObject.cs`

```csharp
public class GameObject : EngineObject, ISerializable
```

### 所有权模型

- **组件列表**：`List<MonoBehaviour>` + 按类型的 `MultiValueDictionary` 缓存
- **层级**：`_parent` + `Children`
- **空间**：内嵌 `[SerializeField] Transform _transform`（**不是**组件）
- **场景**：`WeakReference<Scene>`
- **稳定 ID**：`Guid Identifier`（由 Scene 在加载后恢复）
- **Prefab**：`PrefabAssetId`、`PrefabOverrides`、结构计数

### 关键 API

| 领域 | 方法 |
|------|------|
| 启用 | `Enabled` / `EnabledInHierarchy` |
| 标签层 | `Tag` / `Layer`（经 `TagLayerManager`）、`CompareTag` |
| 层级 | `SetParent(parent, worldPositionStays)`、`IsChildOf`、`GetChildrenDeep` |
| 组件 | `AddComponent<T>()` / `AddComponent(Type)` / `Remove*` / `GetComponent*` / `GetComponentsInParent/Children` |
| Prefab | `IsPrefabInstance`、`ClearPrefabData()` |
| 校验 | `OnValidate()`（递归子物体） |

### AddComponent 流程

1. 处理 `[RequireComponent]`：自动补齐缺失依赖
2. `Activator.CreateInstance` 或挂接已有实例
3. `AttachToGameObject(this)`
4. 写入类型缓存
5. 若已在活动场景：`OnAddedToScene` → `InternalOnEnable`

### SetParent 流程

1. 防环（`IsChildOrSameTransform`）
2. 跨场景时 `oldScene.Remove` / `newScene.Add`
3. 可选保持世界 TRS
4. `HierarchyStateChanged` 传播启用状态
5. UI 相关时标记 `GameCanvas` 脏

### Dispose

先销毁子物体与组件，**主动切断**父子/组件引用图，避免脚本程序集（热重载 ALC）被 GC 钉死。

## 3.5 MonoBehaviour

**文件:** `Prowl.Runtime/GameObject/MonoBehaviour.cs`

```csharp
public abstract class MonoBehaviour : EngineObject, ISerializationCallbackReceiver
```

### 状态字段

- `Guid Identifier`
- `_enabled` / `_enabledInHierarchy`
- `HasStarted` / `HasBeenEnabled`
- `_updateRegistered`（是否在 `SceneComponentRegistry` 中）
- `ShouldExecuteGameplay` → `IsPlaying || [ExecuteAlways]`

### 生命周期虚方法

| 方法 | 时机 |
|------|------|
| `OnAddedToScene` | 进入场景（OnEnable 之前） |
| `OnRemovedFromScene` | 离开场景（OnDisable 之后） |
| `OnEnable` / `OnDisable` | 层级/场景启用变化；用户回调受 `ShouldExecuteGameplay` 约束 |
| `Start` | 首次 Update/FixedUpdate 前，仅一次 |
| `FixedUpdate` | 固定时间步 |
| `Update` / `LateUpdate` | 每帧 |
| `OnRenderCollect` | 每相机收集 Renderable（编辑与 Play 都跑） |
| `DrawGizmos` / `DrawGizmosSelected` | Gizmo 通道 |
| `OnGui(Paper)` | 运行时 Paper UI |
| `OnDispose` | 经 `GameObject.RemoveComponent` 销毁 |

### 内部包装

- `InternalStart/Update/LateUpdate/FixedUpdate`：捕获异常，要求 `ShouldExecuteGameplay`
- `InternalOnEnable`：**总是**注册到场景注册表，再可能调用用户 `OnEnable`
- `InternalOnDisable`：反注册，再可能调用用户 `OnDisable`

这样设计使得 **渲染/Gizmo/GUI 注册与玩法 tick 解耦**：编辑模式下也能预览与绘制。

### 常用属性

| 属性 | 文件 | 作用 |
|------|------|------|
| `[ExecuteAlways]` | `ExecuteAlwaysAttribute.cs` | 编辑模式也执行玩法回调 |
| `[ExecutionOrder(int)]` | `Attributes/ExecutionOrderAttribute.cs` | 注册表排序键 |
| `[RequireComponent(…)]` | `Attributes/RequireComponentAttribute.cs` | 依赖组件；阻止移除最后一份被依赖实例 |
| `HideFlags` | `HideFlags.cs` | `Hide` / `DontSave` / `NoGizmos` 等 |

## 3.6 Transform

**文件:** `Prowl.Runtime/Math/Transform.cs`（命名空间 `Prowl.Vector`）

**不是 MonoBehaviour。** 由 `GameObject` 作为序列化字段持有；`GameObject.Transform` 访问时会重新绑定 `transform.GameObject = this`。

### 本地状态

- `_localPosition` / `_localRotation` / `_localScale`（`[SerializeField]`）
- `_version` 与世界矩阵缓存（`[SerializeIgnore]`）

### API 分类

| 类别 | 成员示例 |
|------|----------|
| TRS | `Position`、`LocalPosition`、`Rotation`、`LocalRotation`、`EulerAngles`、`LocalScale`、`LossyScale` |
| 轴 | `Right` / `Up` / `Forward` |
| 矩阵 | `LocalToWorldMatrix`（懒计算 + 祖先 version）、`WorldToLocalMatrix` |
| 层级 | `Parent`（委托 GO.SetParent）、`Root`、子/兄弟索引 |
| 操作 | `Translate`、`Rotate`、`RotateAround`、`LookAt`、`TransformPoint/Direction/Vector` |
| 原子写 | `SetLocalTransform`、`SetPositionAndRotation`、`SetWorldTransform` |
| 变更检测 | `Version`、`MarkChanged()`、`HasChanged(ref uint)` |

世界矩阵缓存通过本地 `_version` 与父级 `_worldVersion` 失效。UI 使用独立的 `RectTransform` 组件（见 [11-UI系统](11-UI系统.md)）。

## 3.8 与其他系统的耦合点

```
GameObject.AddComponent → MonoBehaviour.Attach
     ↓ (若已在场景)
OnAddedToScene / InternalOnEnable
     ↓
SceneComponentRegistry.Register
     ↓
每帧由 Scene.Update / FixedUpdate / CollectRenderables 驱动
```

1. 组件不能脱离 `GameObject` 存活；场景成员关系挂在 GO 上。
2. 玩法 tick = 注册表成员资格 **且** `ShouldExecuteGameplay`。
3. 渲染/Gizmo/GUI 注册与 Play 状态无关。
4. `Dispose` 切断引用，服务脚本热重载（见 [13-脚本编译与热重载](13-脚本编译与热重载.md)）。
