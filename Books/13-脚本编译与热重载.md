# 13 · 脚本编译与热重载

---

## 13.1 目标

- 用户在 `Assets/**/*.cs` 编写游戏与编辑器脚本
- 通过 `dotnet build` 产出程序集，加载进 **可回收 AssemblyLoadContext（ALC）**
- 支持 **热重载**：尽量不重启进程更新脚本
- 失败时回退到 **进程重启** 并恢复场景

---

## 13.3 编译模型

1. 扫描 `Assets/**/*.cs`
2. 默认编译单元：
   - `{ProjectName}.Game.dll` — 游戏脚本
   - `{ProjectName}.Editor.dll` — `Editor` 文件夹下的编辑器脚本
3. 可选 **Assembly Definition（`.asmdef`）** 多程序集：
   - 引用列表
   - include/exclude 平台
   - unsafe、define 约束
   - 预编译引用、`NoEngineReferences`
4. 为每个单元生成临时 `.csproj`
5. 拓扑排序依赖后执行 `dotnet build`
6. 输出到 `Library/ScriptAssemblies/`

`AssemblyDefinitionImporter` 将 `.asmdef` 纳入资产数据库。

---

## 13.4 热重载流水线

**触发:** `ScriptAssemblyManager.RequestRecompile()`

- **防抖** 约 1 秒
- 仅在窗口聚焦且 **非 Play Mode** 时执行（Play 中结果推迟）

### `TryHotReload` 步骤

```
1. 编译全部用户程序集 (ScriptCompiler.CompileAll)
2. 保存场景 → Library/~autosave.scene (+ 路径/AssetID 侧车)
3. EditorApplication.ReleaseScriptReferences()
     ├─ 清空 Selection / Undo / 当前 Scene 引用
     ├─ 面板 IScriptReloadCleanup
     ├─ 清空 Echo 类型缓存
     └─ 调用 [OnAssemblyUnload] 回调
4. 卸载可回收 ALC（配合多轮 GC，约 ×10）
5. 在新 ALC 中加载新 DLL
6. ReinitializeAfterReload
     └─ 重扫注册表 / 菜单 / 自定义编辑器
7. 恢复场景 + SelectionToken
```

### 回退路径

若 ALC 无法卸载（常见原因：**调试器附加** 钉死可回收 ALC）：

```
进程重启 + CLI:
  --project <path> --restore-scene <autosave>
```

## 13.5 热重载安全规则

### Runtime 侧配合

- `GameObject` / `MonoBehaviour` 的 `Dispose` **主动切断** 父子与组件引用，避免用户脚本类型钉住旧 ALC
- 稳定身份使用 `Guid Identifier`，而非 `InstanceID`
- 缺失类型 → `MissingMonobehaviour` 保底

### 编辑器侧配合

实现 `IScriptReloadCleanup` 的面板/服务在卸载前释放对用户类型的引用。

`ScriptReloadCallbacks` / `[OnAssemblyUnload]` 供扩展注册清理逻辑。

### 禁止 / 慎用

| 做法 | 风险 |
|------|------|
| 静态字段持有 `MonoBehaviour` / 用户类型 | ALC 无法卸载 |
| 事件订阅未取消 | 旧实例复活或泄漏 |
| 跨程序集缓存 `Type` / `MethodInfo` | 指向卸载程序集 |
| 在 Play Mode 期望立即热重载 | 被设计为推迟 |

---

## 13.7 与序列化的交叉

热重载 = **场景往返序列化** + **程序集替换**：

1. Echo 写出完整可保存对象图
2. 旧类型卸载
3. 新类型加载后 Echo 读回
4. 字段/类型不匹配时走缺失脚本或默认值路径

因此脚本字段变更（改名、改类型）可能导致数据丢失——与 Unity 类似，需谨慎或写迁移。

---

## 13.8 调试建议

1. 热重载失败时先 **断开调试器** 再试
2. 检查 Console 中的编译错误（`dotnet build` 输出）
3. 查看 `Library/ScriptAssemblies/` 时间戳是否更新
4. 确认没有静态缓存用户类型
5. 最坏情况依赖 `--restore-scene` 自动恢复
