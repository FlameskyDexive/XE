# 06 · 着色器与材质

---

## 6.1 设计目标

- 提供接近 Unity 的 **自定义着色器 DSL**（Properties / Pass / Tags / Keyword）
- 在 **仅 OpenGL 4.1+** 后端上支持多 Pass、变体缓存
- 用 `PropertyState` 统一材质 / 实例 / 全局属性绑定
- 通过 Tag 与管线协作（Prepass、ShadowCaster、Opaque/Transparent/UI）

---

## 6.2 Shader 资源

**文件:** `Prowl.Runtime/Resources/Shader.cs`

- 持有 `ShaderProperty[]` + `ShaderPass[]`
- 可按 Pass 名或 Tag 查询（`GetPassesWithTag`）
- 加载来源：磁盘路径、或 `BuiltInAssets` 中的默认着色器（`ParseDefault`）

---

## 6.3 ShaderPass

**文件:** `Rendering/Shaders/ShaderPass.cs`

| 成员 | 说明 |
|------|------|
| Name | Pass 名 |
| Tags | 含排序偏移；驱动管线选择 |
| RasterizerState | Cull / Blend / ZTest / ZWrite… |
| Vertex/Fragment 源码 | 字符串 |
| GrabTexture / GrabDepth | 可选屏幕捕获名 |
| 变体缓存 | `TryGetVariantProgram(Dictionary<string,bool> keywords)` |

### 变体编译

1. 组装 `#version 410` + `#define KEYWORD` 前缀
2. `Graphics.CompileProgram`（经渲染线程）
3. 失败时回退 `DefaultShader.Invalid`

`KeywordState` 提供可哈希、可序列化的有序 Keyword 集合；运行时 Pass 变体键目前使用材质 `Dictionary<string,bool>` 拼接。

---

## 6.4 着色器 DSL 与解析器

**文件:** `AssetImporting/ShaderParser.cs`

```
Shader "Name" {
  Properties {
    _MainTex ("Albedo", 2D) = "white" {}
    _Color ("Color", Color) = (1,1,1,1)
    // ...
  }
  Pass {
    Tags { "RenderOrder"="Opaque" "LightMode"="Prepass" }
    GrabTexture "Name"   // 可选
    Blend / Cull / ZTest / ZWrite / ...
    // HLSLPROGRAM 或 GLSL 块，支持 #include
  }
  Fallback "..."
}
```

### 解析流水线

1. 去注释、词法分析
2. `#include` 解析（文件系统 + 内嵌默认）
3. 构建 `ShaderPass`（光栅状态 + 源码）
4. Tag 供管线使用

### 关键 Tag

| Tag | 取值示例 | 作用 |
|-----|----------|------|
| `LightMode` | `Prepass`、`ShadowCaster` | 选择特殊通道 |
| `RenderOrder` | `Opaque`、`Transparent`、`UI` | 合批排序与队列 |

## 6.5 Material

**文件:** `Resources/Material.cs`

| 成员 | 说明 |
|------|------|
| `AssetRef<Shader>` | 着色器引用 |
| `PropertyState _properties` | 当前属性值 |
| `_overrides` | 用户覆盖标记（防止 shader 默认刷新冲掉用户值） |
| `_localKeywords` | 本地 Keyword |
| `GetStateHash()` | 合批用 FNV 风格哈希（脏标记） |

提供各类 `SetFloat/Color/Texture/Matrix…` 与 Keyword 开关。支持从共享模板 `Clone` 出可变实例。

---

## 6.6 PropertyState

**文件:** `Rendering/PropertyState.cs`

类型化字典：float / int / vec / color / matrix / texture / buffer。

两种全局写入方式：

1. 静态 `PropertyState.SetGlobal*` — 简单场景
2. `cmd.SetGlobal*` — **帧中有序** 更安全（推荐）

Executor 在 `PropertyApply` 中结合 program uniform 缓存应用。

---

## 6.7 GlobalUniforms UBO

**文件:** `Rendering/GlobalUniforms.cs`

`GlobalUniformsData`（std140），binding 0：

- View / Proj / 逆矩阵
- 非抖动 VP、上一帧 VP
- 相机位置、投影/屏幕参数、jitter、时间

每绘制调用前绑定。

### 每对象 Uniform（绘制时编码）

- `prowl_ObjectToWorld`
- `prowl_WorldToObject`
- `prowl_PrevObjectToWorld`

## 6.8 运行时 Keyword 一览

| 类别 | Keyword |
|------|---------|
| 几何 | `HAS_NORMALS`、`HAS_TANGENTS`、`HAS_UV`、`HAS_UV2`、`HAS_COLORS`、`HAS_BONEINDICES`、`HAS_BONEWEIGHTS` |
| 蒙皮 | `SKINNED`、`BLENDSHAPES` |
| 实例 | `GPU_INSTANCING` |
| 地形 | `TERRAIN_8_LAYERS`、`TERRAIN_BICUBIC` |
| 色调映射 | `TONEMAP_*`（ACES / Reinhard / Uncharted / Filmic / Melon / AgX） |
| 电影感 | 暗角 / 色散 / 颗粒等开关 |

管线根据网格特性 **自动** 设置几何/蒙皮 Keyword；效果与材质可设置自己的。

## 6.9 内置与默认资源

- `BuiltInAssets` / `DefaultAssets`：默认网格、材质、着色器（确定性 SHA Guid，形如 `$Default:…`）
- `BRDFLutGenerator`：生成 PBR 积分 LUT
- `SMAALookupTextures`：SMAA 查找表（注意朝向与 GL 翻转）

## 6.10 编辑器侧

- `ShaderImporter`：导入 `.shader`
- `MaterialImporter`：导入 `.mat`
- `MaterialAssetEditor` / `MaterialPropertyDrawer`：Inspector 编辑
- 属性变更经 Undo 快照（Echo）

---

## 6.11 编写自定义 Pass 的检查清单

1. 正确设置 `RenderOrder` / `LightMode` Tag
2. 需要深度/法线时依赖 Prepass 输出的全局纹理名
3. 阴影投射 Pass 使用 `LightMode=ShadowCaster`
4. 透明物体注意排序与 `GrabTexture` 成本
5. Keyword 组合爆炸：只在真正分支处定义，避免无意义变体
6. 使用 `GlobalUniforms` 与 `prowl_*` 约定矩阵，勿重复上传
