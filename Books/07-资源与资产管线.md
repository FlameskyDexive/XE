# 07 · 资源与资产管线

---

## 7.1 设计目标

- **GUID 稳定引用**：资产移动/重命名不破坏引用（配合 `.meta`）
- **编辑器与玩家统一接口**：`IAssetDatabase` + `AssetRef<T>`
- **异步加载**：后台线程 + 高/普通优先级队列
- **依赖可追踪**：序列化时收集 Guid 依赖，构建时只打包用到的资源
- **可扩展导入**：属性驱动自定义 Importer

---

## 7.2 核心接口与门面

**文件:** `Prowl.Runtime/AssetDatabase.cs`

```csharp
public interface IAssetDatabase
{
    EngineObject? Get(Guid assetId);        // 可能阻塞 / 触发导入
    EngineObject? GetCached(Guid assetId);  // 永不阻塞
}

public static class AssetDatabase
{
    public static IAssetDatabase? Current { get; set; }
    public static EngineObject? Get(Guid id);       // BuiltIn 优先，再 Current
    public static EngineObject? GetCached(Guid id);
}
```

### 实现

| 实现 | 路径 | 场景 |
|------|------|------|
| `EditorAssetDatabase` | `Prowl.Editor/AssetsDatabase/EditorAssetDatabase.cs` | 扫描、meta、导入、文件监视、依赖图、子资源 |
| `PlayerAssetDatabase` | `Prowl.Runtime/PlayerAssetDatabase.cs` | 松散文件 / ProwlPak / 嵌入 + 二进制 Echo 缓存 + manifest |
| `BuiltInAssets` | `Resources/BuiltInAssets.cs` | 内置默认资源（确定性 Guid） |

## 7.3 AssetRef\<T\>

**文件:** `Prowl.Runtime/AssetRef.cs`

```csharp
public struct AssetRef<T> : IAssetRef, ISerializable where T : EngineObject
```

| API | 行为 |
|-----|------|
| `Res` | 解析资源；异步开启时非阻塞，否则同步 |
| `ResWeak` | 仅缓存 |
| `EnsureLoaded()` | 高优先级阻塞加载 |
| `AssetID` | Guid 身份 |
| `Detach()` | 丢弃本地缓存 |
| `IsExplicitNull` / `IsRuntimeResource` | 显式空 vs 内联运行时对象 |

### 序列化规则

1. 始终写入 `AssetID`
2. Guid 非空：仅引用（在 `DependencySerializationContext` 中追踪依赖）
3. Guid 空 + 存活实例：内联 `Instance` 子树（运行时临时资源）

## 7.4 AssetLoader（异步）

**文件:** `Prowl.Runtime/AssetLoader.cs`

- 单后台线程 `"Prowl Asset Loader"`
- 双队列：**High**（`LoadBlocking` / `EnsureLoaded`）优先于 **Normal**（`Request` / `.Res`）
- 去重：同一 Guid 合并等待者
- 在加载线程调用 `AssetDatabase.Get`；等待者再 `GetCached` 窥探
- `ThreadStatic` 重入：加载线程上的嵌套阻塞加载内联执行（防死锁）
- `AssetLoadingConfig.AsyncEnabled`（默认 true）控制 `AssetRef.Res` 行为
- 关闭时 `Stop()`

### 契约

```csharp
int Version { get; }                    // 缓存失效
void Import(ImportContext ctx);         // SetMainAsset / AddSubAsset
EchoObject DefaultSettings();
bool IsEditorOnlyAsset { get; }         // 玩家打包时排除
```

- **子资源**：主资源 + 确定性 Guid 的 SubAsset（如模型内 Mesh/Material）
- **导入缓存**：Version + 源文件时间戳 + settings 哈希
- **Threaded Asset Loading**：编辑器侧可在后台完成导入

Runtime 侧模型相关：`AssetImporting/ModelImporter.cs`、`ClayBackedImporter.cs`、`LightmapUVGenerator.cs`。

## 7.9 依赖追踪

序列化时：

```
DependencySerializationContext
  └─ 收集所有 AssetRef / 带 AssetID 的 EngineObject Guid
```

编辑器 `DependencyGraph`：

- 正向：A 依赖哪些
- 反向：谁依赖 A（删除/移动时提示与失效）

构建 `AssetCollector`：

- 从启用场景出发 BFS/DFS
- `AssetMode.DependenciesOnly` 仅打包用到的；`AllAssets` 打包全部
- 跳过 `IsEditorOnlyAsset`

---

## 7.10 最佳实践

1. 字段用 `AssetRef<T>` 而非直接 `T` 引用磁盘资源
2. 需要保证已加载时用 `EnsureLoaded()` 或同步路径
3. 自定义 Importer 务必递增 `Version` 以失效旧缓存
4. 不要手改 `.meta` 的 Guid（除非你知道依赖图后果）
5. 热路径避免每帧 `AssetDatabase.Get`；缓存 `Res` 结果
