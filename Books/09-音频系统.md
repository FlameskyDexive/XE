# 09 · 音频系统

---

## 9.1 概览

- **后端**: MiniAudio（原生包装于 `Prowl.Runtime/Audio/Native/`）
- **核心**: 静态 `AudioContext`（`Audio/AudioContext.cs`）
- **组件**: `AudioSource`、`AudioListener`（`Components/Audio/`）
- **效果**: `IAudioEffect` 链（Delay / Distortion / Filter / Phaser / Reverb…）

初始化发生在 `Game.Run` 的窗口 Load 阶段；Headless 模式通常跳过音频。

---

## 9.2 AudioContext

```csharp
AudioContext.Initialize(sampleRate, channels, periodSize, deviceInfo?);
AudioContext.Deinitialize();
AudioContext.Update();
```

| 成员 | 说明 |
|------|------|
| `SampleRate` / `Channels` | 设备格式 |
| `MasterVolume` | 主音量 |
| `DeltaTime` | 音频侧 delta |
| `GetDevices()` | 枚举设备 |
| `GetOutputBuffer` | 输出缓冲访问 |
| `DataProcess` 事件 | 设备回调钩子 |

辅助：`DeviceInfo`、`AudioBuffer`（环形）、`ConcurrentList<T>`。  
枚举：`AttenuationModel`、`PanMode`。

设备回调路径中完成混音与效果处理，主线程通过组件 API 提交播放指令。

---

## 9.3 AudioListener

**文件:** `Components/Audio/AudioListener.cs`

- 每帧 `Update` 将 `Transform` 的位置与朝向同步到原生 Listener
- 在 `OnEnable` / `OnDisable` 管理原生句柄生命周期
- 场景中通常只需一个活动 Listener

---

## 9.4 AudioSource

**文件:** `Components/Audio/AudioSource.cs`（`MonoBehaviour, ISerializable`）

### 播放

| API | 说明 |
|-----|------|
| `Clip` | `AudioClip` 资源 |
| `PlayOnStart` / `Loop` | 自动播放与循环 |
| `Play()` / `Stop()` | 控制 |
| `PlayOneShot` | 一次性 |
| `PlayProcedural()` | 程序化源 |

### 混合与空间

| 属性 | 说明 |
|------|------|
| `Volume` / `Pitch` / `Pan` / `PanMode` | 基础混合 |
| `Spatial` | 是否 3D |
| `DopplerFactor` | 多普勒 |
| `MinDistance` / `MaxDistance` | 衰减距离 |
| `AttenuationModel` | 衰减模型 |
| `IsPlaying` / `Cursor` / `Length` | 状态 |

### 效果链

```csharp
AddEffect(IAudioEffect)
RemoveEffect(...)
ClearEffects()
EffectCount
```

### 事件

`End`、`Process`、`Read`

多普勒使用 `GetCalculatedVelocity()`（由 Transform 位移推导）。

## 9.5 效果系统

**接口:** `Audio/Effects/IAudioEffect.cs`

```csharp
void OnProcess(NativeArray<float> framesIn, uint frameCountIn,
               NativeArray<float> framesOut, ref uint frameCountOut, uint channels);
void OnDestroy();
```

| 效果 | 路径 | 主要参数 |
|------|------|----------|
| `DelayEffect` | `Effects/DelayEffect.cs` | Wet/Dry/Decay、DelayInFrames/Seconds |
| `DistortionEffect` | | Drive、Range、Blend、Volume |
| `FilterEffect` + `Filter` | | FilterType、Frequency、Q、GainDB |
| `PhaserEffect` + `Phaser` | | 相位类 DSP |
| `ReverbEffect` + `Reverb` | | 混响 DSP |
| `Envelope` / `ADSR` | | Attack/Decay/Sustain/Release |
| `DelayLine` | | 构建块 |

自定义效果实现 `IAudioEffect` 并挂到 Source 即可进入处理链。

## 9.7 数据流

```
设备回调 (AudioContext.DataProcess)
  └─ 各活动 AudioSource
        ├─ 解码/读取 Clip 或程序化数据
        ├─ 效果链 OnProcess（依次）
        ├─ 相对 AudioListener 的空间化 / 衰减 / 多普勒
        └─ 混入主总线 → MasterVolume → 设备输出
```

主线程：`AudioContext.Update()` + 组件 `Update` 同步 Listener/Source 参数。

---

## 9.8 使用建议

1. 保证场景中有启用的 `AudioListener`
2. 3D 音效开启 `Spatial` 并合理设置 Min/MaxDistance
3. 效果链在音频线程执行——`OnProcess` 中避免托管分配与锁竞争
4. 短音效用 `PlayOneShot`；长音乐注意 Loop 与资源生命周期
5. 结合 InputAction 做交互音效时可参考 `Samples/AudioDemo`
