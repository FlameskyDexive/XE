# 04 · 场景与生命周期

## 4.1 Scene 资源

**文件:** `Prowl.Runtime/Resources/Scene.cs`

```csharp
[CreateAssetMenu("Scene", Extension = ".scene")]
public class Scene : EngineObject, ISerializationCallbackReceiver
```

### 内置场景管理器

```csharp
public static Scene? Current { get; private set; }
public static event Action? OnSceneLoaded;
public static void Load(Scene scene);  // 禁用+Dispose 旧场景，启用新场景
public static void Unload();
```

默认单活动场景。高级宿主可自行持有多个 `Scene` 实例，直接调用 `Enable` / `Disable` / `Update`。

### 场景内容

| 成员 | 说明 |
|------|------|
| `_allObj` + HashSet | 扁平注册所有 GO（引用相等） |
| `Physics` | 每场景一个 `PhysicsWorld` |
| `ComponentRegistry` | 内部 tick 注册表 |
| `Fog` / `Ambient` / `Skybox` | 环境渲染参数 |
| `BakedLighting` / `LightmapBake` / `ProbeVolume` | 烘焙光照与探针 |

### 查询 API

`AllObjects`、`ActiveObjects`、`RootObjects`、`ActiveRootObjects`、`SaveableObjects`（尊重 HideFlags）、`FindObjectsOfType<T>`、`FindObjectByID`、`FindObjectByIdentifier`。

## 4.2 加入 / 移除对象

### `AddObject` / 递归

1. 深度遍历子物体
2. 设置 `obj.Scene`
3. 对每个组件：`OnAddedToScene`；若场景活动且组件启用 → `InternalOnEnable`
4. 注册进扁平列表

### `RemoveObject`

反向：`InternalOnDisable` → `OnRemovedFromScene` → 清空 Scene 引用。

场景级 `Enable` / `Disable` 会遍历所有 GO/组件调用对应 Internal 启停。

## 4.3 SceneComponentRegistry

**文件:** `Prowl.Runtime/GameObject/SceneComponentRegistry.cs`

每场景一份，负责 **有序、安全** 地驱动组件回调。

### 能力检测

对每个 `Type` 反射一次，记录是否重写了：

`Start`、`Update`、`LateUpdate`、`FixedUpdate`、`OnRenderCollect`、`DrawGizmos`、`OnGui`。

未重写的通道 **不进入列表**，降低每帧开销。

### 通道

| 通道 | 驱动方 |
|------|--------|
| Start | `Update` / `FixedUpdate` 开头 |
| Update / LateUpdate | `Scene.Update` |
| FixedUpdate | `Scene.FixedUpdate`（Physics 之后） |
| RenderCollect | 相机收集 |
| DrawGizmos | 编辑/调试 |
| OnGui | Paper 帧 |

### 排序与安全

- 按 `[ExecutionOrder]` 稳定排序（相等时保持插入序）
- **脏标记 + 快照**：回调中途启用/禁用其他组件时，当前迭代使用快照，下一帧再重建有序列表
- 启用时 `Register`，禁用时 `Unregister`

## 4.4 每帧场景 API

| 方法 | 行为 |
|------|------|
| `Update()` | Start → Update → LateUpdate → `Flush()` 销毁队列 |
| `FixedUpdate()` | Start（幂等）→ `Physics.Update()` → FixedUpdate → Flush |
| `CollectRenderables(cam, …)` | 注册表渲染收集 |
| `DrawGizmos()` / `OnGui(paper)` | 对应通道 |
| `Render(RenderTexture?)` | 按 `Camera.Depth` 排序，逐相机走 `RenderPipeline` |

`Flush` 负责真正销毁已标记删除的 GameObject，避免在遍历中修改集合。

## 4.5 完整一帧时间线（Play Mode）

```
1. Time.Update()，压入 TimeStack
2. Input Action 更新（可用 unscaled dt）
3. 固定步循环（若 ShouldRunGameplay）:
     Scene.FixedUpdate
       ├─ Start（尚未 Start 的组件）
       ├─ PhysicsWorld.Update
       └─ FixedUpdate 回调
4. Scene.Update
     ├─ Start
     ├─ Update
     ├─ LateUpdate
     └─ Flush
5. 渲染
     ├─ 清屏 CB
     ├─ ShadowAtlas 初始化/清空
     └─ Scene.Render
          └─ 每相机 DefaultRenderPipeline
               └─ OnRenderCollect → Cull → Draw → 后处理
6. Paper GUI：Scene.OnGui
7. RT 池维护 + 延迟 GPU Dispose
8. 若单步：清除 StepRequested，恢复 IsPaused
```

## 4.6 组件生命周期时序图

```
[创建 GO / AddComponent]
        │
        ▼
  (加入 Scene)
        │
        ├─ OnAddedToScene
        └─ InternalOnEnable
              ├─ 注册 Registry
              └─ OnEnable? (若 ShouldExecuteGameplay)
        │
        ▼
  (首帧 tick)
        └─ Start  (仅一次)
        │
        ├── FixedUpdate  (0..N 次/帧)
        ├── Update
        └── LateUpdate
        │
        ▼
  (相机渲染)
        └─ OnRenderCollect  (每相机)
        │
        ▼
  (禁用 / 移出场景)
        └─ InternalOnDisable
              ├─ 反注册
              └─ OnDisable?
        └─ OnRemovedFromScene  (若移出)
        │
        ▼
  (销毁)
        └─ OnDispose / Dispose
```

## 4.7 环境与渲染相关场景状态

场景持有影响全局着色的环境数据：

- **雾 (Fog)**、**环境光 (Ambient)**
- **天空盒 (Skybox)**：程序天空 / 纯色 / 渐变 / 自定义材质
- **烘焙光照与 Light Probe**：由编辑器 Lightmapping 服务写入，运行时 `LightmapBinding` / `LightProbeVolume` 消费

这些在 `DefaultRenderPipeline` 中作为全局 Uniform 上传。

## 4.8 序列化时的身份恢复

场景序列化（详见 [15-序列化与Prefab](15-序列化与Prefab.md)）采用「反序列化生成临时 Guid → `OnAfterDeserialize` 写回稳定 Guid → `Add(obj)` 重建注册」：

1. `OnBeforeSerialize`：拍平对象，捕获 GO/组件 Identifier 并行数组
2. 反序列化 GO 树（组件先于子物体，保证 `$id` 图引用顺序）
3. `OnAfterDeserialize`：还原 Guid，再 `Add` 触发生命周期

缺失脚本类型会变成 `MissingMonobehaviour` 持有原始 Echo 子树，后续可在类型恢复后回补。

## 4.9 设计要点小结

1. **单场景默认，多场景可选**——`Current` 是便利而非硬限制。
2. **注册表 + 反射能力位**——只调度实际重写的回调。
3. **快照迭代**——生命周期回调中修改组件集合安全。
4. **双重门控**——Registry 成员资格 × `ShouldExecuteGameplay`。
5. **物理绑定场景**——每个 Scene 自有 `PhysicsWorld`，加载/卸载时随场景生命周期。
