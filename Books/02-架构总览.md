# 02 · 架构总览

---

## 2.1 分层结构

Prowl 的代码按「可独立运行的 Runtime」与「依赖 Runtime 的 Editor」清晰切分：

```
┌─────────────────────────────────────────────────────────────┐
│                     Prowl.Editor                            │
│  面板 / Undo / 选择 / 资源导入 / 脚本编译 / 构建 / 本地化        │
└───────────────────────────┬─────────────────────────────────┘
                            │ ProjectReference
┌───────────────────────────▼─────────────────────────────────┐
│                     Prowl.Runtime                           │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐       │
│  │ Game/App │ │ Scene/GO │ │ Graphics │ │ Physics  │       │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘       │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐       │
│  │  Audio   │ │  Input   │ │  Assets  │ │   UI     │       │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘       │
└───────────────────────────┬─────────────────────────────────┘
                            │ NuGet / 原生库
        Silk.NET · Jitter2 · MiniAudio · Prowl.Echo/Paper/Vector/...
```

**关键约束：**

1. **Runtime 不引用 Editor**。玩家构建、Samples、服务器式 headless 循环都只需 Runtime。
2. **Editor 通过虚方法与全局标志**（`Application.IsEditor` / `IsPlaying`）与 Runtime 协作，而不是侵入组件基类。
3. **资源身份**（`AssetID: Guid`）在编辑器与玩家两侧统一，实现方通过 `IAssetDatabase` 策略切换。

## 2.2 对象与身份模型

```
EngineObject (抽象基类, IDisposable)
├── GameObject
├── MonoBehaviour  (+ 各具体组件)
└── 资源类 (Scene, Mesh, Material, Shader, Texture*, PrefabAsset, ...)
```

| 身份字段 | 生命周期 | 用途 |
|----------|----------|------|
| `InstanceID` | 进程内单调递增 | 运行时引用、调试 |
| `Identifier` (GO/Component) | 随场景存盘 | Undo、Selection 恢复、热重载后回挂 |
| `AssetID` | 资源数据库主键 | 跨会话引用；非空则序列化为引用而非内联 |

判断有效性：`obj.IsValid()` / `IsNotValid()`（null 或已 `Dispose`）。

## 2.3 主循环数据流

### 窗口模式（`Game.Run`）

```
Window.Init
  └─ Load: Audio / BuiltInAssets / Game.Initialize()
MainLoop:
  Graphics.BeginFrame()
  DoEvents
  Update  → Game.SimulationStep(dt)
  Render  → Game.OnRender → Scene.Render → RenderPipeline
  GUI     → Paper + Game.OnGui
  EndFrameAndWait() → 渲染线程 SwapBuffers
Closing → Scene.Unload / Audio 释放
```

### 单帧模拟（`SimulationStep`）

```
fixedTimeAccumulator += delta
if Application.ShouldRunGameplay:
    while accumulator >= FixedDeltaTime (有上限):
        Scene.Current.FixedUpdate()   # Start残留 → Physics → FixedUpdate
    OnUpdate → Scene.Update()         # Start → Update → LateUpdate → Flush
else:
    仅钳制 accumulator（暂停/非 Play）
消费 StepRequested（单步）
```

### 渲染帧

```
主线程编码 CommandBuffer
  → Graphics.Submit / SubmitAndWait
渲染线程 (独占 GL Context)
  → CommandExecutor.Execute → OpenGL
  → 帧末 SwapBuffers
```

详见 [05-图形与渲染管线](05-图形与渲染管线.md)。

## 2.4 线程模型

| 线程 | 职责 |
|------|------|
| **主线程** | 输入、模拟、场景逻辑、编码 GPU 命令、Paper UI |
| **GL 渲染线程** | 持有 OpenGL Context；执行 CommandBuffer；SwapBuffers |
| **资源加载线程** | `AssetLoader` 单后台线程，处理异步 `AssetRef` 解析 |
| **脚本编译** | 编辑器内 `dotnet build` 在后台任务中执行 |

**硬规则：**

- 主线程在启动渲染线程后 **不再持有 GL Context**。
- 所有 GL 状态变更必须通过 `CommandBuffer`（`Submit` 或 `SubmitAndWait`）。
- 资源在帧中被替换时走 `Graphics.DeferDispose`，帧末统一释放，避免 use-after-free。

## 2.5 全局服务与模式

| 全局入口 | 角色 |
|----------|------|
| `Application` | 静态运行模式标志（IsPlaying / IsEditor / IsPaused …） |
| `Time` | 时间栈与固定步长常量 |
| `Scene.Current` | 当前活动场景（可高级用法多场景手动管理） |
| `AssetDatabase.Current` | 当前 `IAssetDatabase` 实现 |
| `Input` | 输入 Handler 栈 + Action Map 注册 |
| `Graphics` | GL 门面、命令提交、资源工厂 |
| `AudioContext` | 音频设备与主总线 |
| `Debug` | 日志与断言 |

常用设计模式：

- **Template Method**：`Game` 虚钩子（`Initialize` / `OnUpdate` / `OnRender` …）
- **Registry**：`SceneComponentRegistry`、Editor 各类属性驱动注册表
- **Strategy**：`IAssetDatabase`（Editor / Player / BuiltIn）
- **Command**：`CommandBuffer` 录制-回放
- **Handle / Ref**：`AssetRef<T>`
- **Prototype**：`PrefabAsset.Instantiate`

## 2.6 编辑器 vs 玩家边界

```
Application.IsEditor     // 是否在编辑器进程
Application.IsPlaying    // 是否处于 Play Mode（玩家恒为 true）
Application.ShouldRunGameplay  // IsPlaying && (!Paused || Step)
MonoBehaviour.ShouldExecuteGameplay
  = Application.IsPlaying || [ExecuteAlways]
```

| 行为 | 编辑模式 | Play Mode / 玩家 |
|------|----------|------------------|
| `Update` / `FixedUpdate` | 仅 `[ExecuteAlways]` | 正常执行 |
| `OnRenderCollect` / Gizmo / OnGui | 始终 | 始终 |
| 物理步进 | 通常不跑（取决于 ShouldRunGameplay） | 跑 |
| 热重载 / Undo | 可用 | Play 中 Undo 禁用；重载推迟 |
