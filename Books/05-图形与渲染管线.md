## 5.2 渲染线程协议

**源文件:** `Graphics/Graphics.cs`、`Window.cs`

1. 主线程在启动后 **不再持有** GL Context。
2. `Graphics.StartRenderThread()` 清空主线程 Context，启动 `"Prowl GL Render Thread"`，在其上 `MakeCurrent()` 并终身持有。
3. 工作队列：`BlockingCollection<CBJob>`
   - `Submit(CommandBuffer)` — 异步
   - `SubmitAndWait(CommandBuffer)` — 阻塞直到执行完，渲染线程异常会重新抛出（读回、编译、FBO 检查）
   - 帧末哨兵 — `SwapBuffers` + 信号 `s_renderFrameDone`
4. 帧节奏：

```
Graphics.BeginFrame()          // 武装 frameDone
DoEvents / Update / Render     // 主线程编码并 Submit
Graphics.EndFrameAndWait()     // 推哨兵，等待 SwapBuffers
```

5. 渲染线程持续排空队列（不限于帧内），帧间的创建/SubmitAndWait 也能及时处理。
6. **Headless**：`Graphics.IsHeadless`（`GL == null`）时 Submit 为空操作并归还池。
7. **延迟释放**：帧中替换的资源走 `Graphics.DeferDispose`，`FlushDeferredDisposes()` 在帧后执行。
8. `LastFrameWaitMs` 衡量主线程等待渲染线程的时间（瓶颈信号）。

## 5.3 CommandBuffer 设计

**源文件:**  
`Graphics/Commands/CommandBuffer.cs`、`CommandOpcode.cs`、`CommandExecutor.cs`、`CommandBufferPool.cs`、`PropertyStatePool.cs`、`TransientStore.cs`

### 录制模型

- 池化：`Graphics.GetCommandBuffer(name)`
- 单生产者编码；按 **提交顺序** 执行
- 后备存储：
  - `_stream`（`byte[]`）— opcode 头 + 定长载荷
  - `_objects` — 对象引用（`ushort` 索引）
  - `_store`（`TransientStore`）— 大块数据（矩阵数组、上传字节）
  - 从 `PropertyStatePool` 租用的 `PropertyState` 快照，保证编码时捕获不被后续修改污染

### Opcode 类别

- 渲染目标 / 视口 / 裁剪 / 清屏 / FBO blit
- 光栅状态、绑定着色器
- 属性绑定：材质、实例、全局（全局在 **执行时** 才应用，保证 CB 中途 Set/Clear 有序）
- 逐 Uniform 设置
- Buffer/Texture/VAO/FBO 创建、上传、销毁；着色器编译
- 绘制：`DrawIndexed`、`DrawIndexedInstanced`、`DrawArrays`
- Debug sample 标记

### 编码糖（展开为底层 op，无独立 executor case）

```csharp
cmd.DrawMesh(mesh, material, pass, model, properties);
cmd.Blit(source, dest, material, pass, ...);
```

### 典型用法

```csharp
using var cmd = Graphics.GetCommandBuffer("My Pass");
cmd.SetRenderTarget(rt.frameBuffer);
cmd.ClearRenderTarget(...);
cmd.DrawMesh(mesh, material);
Graphics.Submit(cmd); // 所有权转移；执行后归还池
```

### Executor

- **唯一** 发出 Silk.NET OpenGL 绘制/状态调用的地方（资源构造路径除外）
- 镜像状态：当前 program、draw/read FBO、VAO、光栅状态、粘滞的 material/instance PropertyState
- 每次绘制前 `PrepareDraw()`：
  - 绑定 `GlobalUniforms` UBO（binding 0）
  - 应用 globals + material（含 shader 默认）+ instance
  - 冲刷延迟的直接纹理绑定
- 通过镜像跳过冗余绑定

## 5.4 RenderPipeline 基类

**文件:** `Rendering/RenderPipeline.cs`

### 核心抽象

- **`IRenderable`**：材质、层、位置、子网格、`GetRenderingData` / `GetCullingData`、可选 `GetWorldToObjectMatrix`
- **`IRenderableLight`** + `ForwardLightData` + `LightType`（Directional / Spot / Point）
- **`CameraSnapshot`**：冻结本帧相机/视图/投影/视锥

### 内建能力

| 能力 | 说明 |
|------|------|
| 剔除 | `CullRenderables`：视锥 + LayerMask；世界 AABB 按帧缓存 |
| 排序 | FrontToBack / BackToFront（距离平方） |
| 合批 | 按 `(material state hash, pass index, mesh)` 分组；同 Tag 多 Pass；排序键含 Tag 偏移（如 Transparent+1000） |
| 网格 Keyword | `HAS_NORMALS`、`HAS_TANGENTS`、`HAS_UV`、`HAS_UV2`、`HAS_COLORS`、`HAS_BONEINDICES`、`HAS_BONEWEIGHTS`、`SKINNED`、`BLENDSHAPES`、`GPU_INSTANCING` |
| GrabTexture | Pass 前可选屏幕捕获（含深度） |
| 运动追踪 | 上一帧模型矩阵 → `prowl_PrevObjectToWorld` |
| 全局 UBO | `SetupGlobalUniforms` / `AssignCameraMatrices` |

## 5.5 DefaultRenderPipeline 通道顺序

**文件:** `Rendering/DefaultRenderPipeline.cs`

入口：

```
Game.OnRender → Scene.Render()
  → 相机按 Camera.Depth 排序
  → cam.Pipeline ?? DefaultRenderPipeline.Default
  → pipeline.Render(camera, RenderingData)
```

| 步骤 | 内容 | 备注 |
|------|------|------|
| 0 Setup | HDR 标志、按 Stage 收集 ImageEffect、`camera.UpdateRenderData()` | |
| 1 PreCull | `effect.OnPreCull`（TAA jitter、GTAO dither） | |
| 2 Globals | `CameraSnapshot`、`SetupGlobalUniforms` | |
| 3 Collect+Cull | `CollectRenderables`、可选编辑器网格、视锥剔除 | |
| 4 PreRender | `effect.OnPreRender` | |
| 5 Lights+Shadows | `SceneLightSystem.Reconcile` → 清图集 → `RenderShadows` → 重设相机矩阵 → `UploadGlobalUniforms` | 每 cascade/face 独立 CB |
| 6 Fog/Ambient | 场景雾与环境光作全局量 | |
| 7 Prepass | MRT：深度 + 视空间法线 + motion/rough/metal | Tag: `LightMode=Prepass` |
| 8 G-Buffer 全局 | `_CameraDepthTexture` 等 | |
| 9 Depth copy | Prepass 深度 → 颜色 RT | |
| 10 Clear/Sky | 天空盒 / 纯色 / 渐变 / 无 | |
| 11 Opaques | Forward PBR，可 GrabTexture | Tag: `RenderOrder=Opaque` |
| 12 AfterOpaques FX | GTAO、SSR 等 | Stage `AfterOpaques` |
| 13 Transparents | 从后往前 | Tag: `RenderOrder=Transparent` |
| 14 World UI | 世界空间 Canvas 进场景色 | |
| 15 PostProcess FX | Bloom、TAA、Tonemap、AA… | Stage `PostProcess`；可 `ReplaceSceneColor` |
| 16 Gizmos + blit | 可选 Gizmo，blit 到相机目标 | |
| 17 Overlay UI | 屏幕空间 UI 到最终目标 | |
| 18 Reset FBO | 回到后缓冲 + 全视口 | 供 Paper/编辑器 UI |
| 19 Cleanup | 保存 prev VP、`OnPostRender`、释放临时 RT | |

**临时目标：**

- `colorRT` — HDR（`Short4`）或 LDR（`Color4b`）+ 深度
- `prepass` — MRT 2 色 + 深度

**天空盒模式**（`Scene.Skybox`）：Procedural / SolidColor / Gradient / Material。

## 5.6 阴影系统

### ShadowAtlas

**文件:** `Rendering/ShadowAtlas.cs`

- 单张深度 `RenderTexture`：设备允许则 8192²，否则 4096²
- 开启深度比较采样（硬件 2×2 PCF + 线性过滤）
- Guillotine 自由矩形装箱（`ReserveTiles`，Best-Short-Side-Fit）
- 每帧 `Clear` 重置空闲列表 + CB 深度清空

### SceneLightSystem

**文件:** `Rendering/SceneLightSystem.cs`

- 每场景一份（管线弱表）
- **静态 + 动态 Light BVH** 管理局部光（点/聚光）；GPU 纹理供片元无栈遍历
- **一个平行光** 特判（不进 BVH）
- 每帧最多 **4 个** 带阴影的局部光（距相机最近）
- 烘焙光（`LightBakeMode.Baked`）排除出实时集合
- `Reconcile` → 增删重构 → 选 caster → 同步纹理
- `RenderShadows` → 平行光 cascade + 选中局部光
- `UploadGlobalUniforms` — 一张 CB 上传 BVH、平行光 cascade、局部阴影槽、图集

### 各光阴影布局

| 光类型 | 布局 |
|--------|------|
| `DirectionalLight` | 1/2/4 线性 cascade；图集 tile；纹素对齐正交投影；`LightMode=ShadowCaster` |
| `PointLight` | 6 面，3×2 图集布局 |
| `SpotLight` | 单透视 tile |

**重要：** 每个 face/cascade 使用 **独立 CommandBuffer**，因为 `AssignCameraMatrices` 会改共享 GlobalUniforms UBO；共用一个 CB 会导致所有 face 用最后一组矩阵绘制。

## 5.7 后处理链

**基类:** `ImageEffect`（挂在 `Camera.Effects`）  
**阶段:** `RenderStage.AfterOpaques` | `PostProcess`  
**上下文:** `RenderContext` — 提供 DepthNormals、运动向量、SceneColor、尺寸；仅 PostProcess 可 `ReplaceSceneColor`

### 钩子顺序

`OnPreCull` → `OnPreRender` → `OnRenderEffect` → `OnPostRender`（移除/禁用时 `OnDisable`）

### 内置效果（`Rendering/Image Effects/`）

| 效果 | Stage | 说明 |
|------|-------|------|
| `GTAOEffect` | AfterOpaques | GTAO；时域；半/四分分辨率 |
| `ScreenSpaceReflectionEffect` | AfterOpaques | 深度/法线/粗糙金属 |
| `VolumetricFogEffect` | 图像效果 | 体积雾 / 雾光 |
| `BloomEffect` | PostProcess | dual-filter 下采样/上采样 |
| `TonemapperEffect` | PostProcess | AgX/ACES 等 Keyword；HDR→LDR |
| `TAAEffect` | PostProcess | PreCull 中 Halton 抖动；运动向量 resolve |
| `FXAAEffect` / `SMAAEffect` | PostProcess | 空间 AA |
| `MotionBlurEffect` | PostProcess | 运动模糊 |
| `BokehDepthOfFieldEffect` | PostProcess | 景深 |
| `AutoExposureEffect` | PostProcess | 曝光适应 |
| `CinematicEffects` | PostProcess | Uber：暗角、色散、颗粒、调色 |

各效果租用并提交自己的 CB。

## 5.8 相机与几何渲染器

### Camera（`Components/Camera.cs`）

- ClearFlags、CullingMask、FOV/正交、近远裁、Depth 排序
- 可选自定义 `RenderPipeline` / `Target`、HDR、`RenderScale`
- 投影可被 TAA 抖动；`NonJitteredProjectionMatrix` 保持干净
- `PreviousViewProjectionMatrix` 由管线末尾写入
- `Effects: List<ImageEffect>`

### MeshRenderer

- 网格 + 每子网格材质
- 每个子网格发射一个 `MeshRenderable`
- 光照贴图索引 / scale-offset 经 `LightmapBinding.Fill`
- 缓存 `PropertyState`（`_ObjectID`、morph 计数）

### SkinnedMeshRenderer

- 骨骼路径解析到 Transform；蒙皮矩阵写入 **float 纹理**（每骨 4 个 RGBA32F texel）
- 用 `Transform.Version` 之和做脏检测
- BlendShape 经 morph weight 纹理
- 发射 `SkinnedMeshRenderable`，AABB 由骨骼推导
- Keyword：`SKINNED` / `BLENDSHAPES`

### GPU Instancing

- `InstanceData`：模型列、颜色、自定义 Float4
- `InstancedMeshRenderable` + `CreateBatched`
- 管线：`mesh.EnsureInstanceVAO` + `UpdateBuffer` + `DrawIndexedInstanced` + `GPU_INSTANCING`

## 5.9 地形渲染

**路径:** `Components/Terrain/`

- GPU 实例化 patch：共享网格，VS 采样 heightmap
- `TerrainQuadtree` LOD
- Keyword：`TERRAIN_8_LAYERS`、`TERRAIN_BICUBIC`
- 属性：heightmap、splatmap、holes、分层 albedo/normal/tiling/PBR、尺寸/高度/矩阵
- **草**：密度原型、距离衰减、实例化叶片
- **树**：距离剔除实例网格
- 编辑器笔刷预览 Uniform

## 5.10 粒子渲染

**路径:** `Components/ParticleSystem/`

- CPU 模块模拟：发射、生命周期 size/color/rotation/velocity、碰撞、UV、光
- 渲染：公告板四边形 + `InstancedMeshRenderable.CreateBatched`
- 可选每粒子点光：`ParticleLightProxy` 稳定代理，使 LightBVH 做 refit 而非 rebuild

## 5.11 光照模型摘要

- **平行光 + cascade 阴影** 作为全局 Uniform
- **局部光** 经双 Light BVH 纹理在片元着色器中遍历
- 最多 4 个带阴影局部光走图集
- 着色：PBR Metallic + BRDF LUT（`BRDFLutGenerator`）
- 烘焙：Lightmap + Light Probe（`SphericalHarmonicsL2`）
