# 11 · UI 系统

Prowl 存在 **两套 UI 技术栈**，服务不同场景：

| 栈 | 用途 | 技术 |
|----|------|------|
| **游戏内 UI** | 玩家 HUD、菜单、世界空间 UI | GameObject + `RectTransform` + UI 组件 |
| **Paper / Origami** | 编辑器壳、运行时调试 OnGui、部分工具 | Immediate Mode（Paper）+ 组件库（Origami） |

---

## 11.1 游戏内 UI 架构

**路径:** `Prowl.Runtime/Components/UI/`

### 核心概念

```
GameCanvas / UISurface
  └─ 层级下的 UI GameObject
        ├─ RectTransform   (布局)
        ├─ UIImage / TextComponent / ...
        ├─ Selectable 派生 (Button, Slider, ...)
        └─ LayoutGroup 等布局组件
```

| 类型 | 角色 |
|------|------|
| `RectTransform` | 锚点、轴心、尺寸、偏移；与 3D `Transform` 并存 |
| `GameCanvas` | 画布根；脏标记与缩放模式 |
| `UISurface` | 渲染表面抽象 |
| `UIContext` | UI 上下文状态 |
| `UIBehaviour` | UI 组件基类 |
| `CanvasGroup` | 组透明度/交互开关 |
| `RectMask` | 矩形裁剪（着色器 Uniform，非 GPU scissor） |

### 渲染模式

- `RenderMode`：屏幕空间 / 世界空间等
- `ScaleMode` / `ScreenMatchMode`：分辨率适配

### 嵌入渲染管线

**文件:** `UIRenderTree.cs`、`UIRenderItem.cs`、`UIMeshBuilder.cs`

1. 收集：`UIRenderTree.CollectFor(scene, surface)`
2. 绘制：材质 Tag `RenderOrder=UI`，经 `cmd.DrawMesh`
3. **世界 UI**：半透明之后，使用相机矩阵
4. **覆盖 UI**：最终 blit 之后，屏幕正交（原点左下）
5. 场景视图中屏幕空间 Canvas 会以世界形式绘制以便编辑
6. 排序键：层级 `SortKey`；Item 池化

详见 [05-图形与渲染管线](05-图形与渲染管线.md) 通道表第 14、17 步。

## 11.2 布局系统

**路径:** `Components/UI/Layout/`

| 组件 | 作用 |
|------|------|
| `LayoutElement` | 首选/最小/灵活尺寸 |
| `LayoutGroup` | 基类布局组 |
| `GridLayoutGroup` | 网格 |
| `ContentSizeFitter` | 内容自适应 |

布局通过脏标志（`UIDirtyFlags`）延迟重算，避免每帧全树遍历。

`GameObject.RectTransform` / `EnsureRectTransform()` 用于获取或创建 UI 变换。

## 11.3 交互与事件

**路径:** `Components/UI/Input/`

- `EventSystem`：分发指针/提交/取消等

`IPointerEnter/Exit/Down/Up/ClickHandler`、`IBeginDrag/Drag/EndDrag/DropHandler`、`IScrollHandler`、`ISelect/DeselectHandler`、`ISubmit/CancelHandler`、`IMoveHandler`

## 11.4 Paper Immediate Mode UI

**运行时集成:** `GUI/PaperRenderer.cs`  
**库:** [Prowl.Paper](https://github.com/ProwlEngine/Prowl.Paper)

- 在 `Game.OnGui` / `Scene.OnGui` / 组件 `OnGui(Paper)` 中调用
- 编辑器整壳基于 Paper + **Origami**（PropertyGrid、常用控件）
- 矢量与文字底层可走 Quill / Scribe

典型用途：

- 编辑器面板与停靠布局（DockSpace）
- 运行时调试 HUD
- 工具弹窗

游戏正式 UI 更推荐 **GameObject UI**（可序列化、可 Prefab、可世界空间）。

## 11.6 字体与文本

- `FontAsset` + `FontImporter`（ttf/otf）
- `UIFontSystem` 管理运行时字形
- `TextMeshComponent`：3D 世界文本网格
- 编辑器侧 `TextComponentEditor` / `TextMeshComponentEditor`

---

## 11.7 设计建议

1. **屏幕 HUD**：Overlay Canvas + RectTransform 布局组
2. **世界铭牌/交互提示**：World Space Canvas，注意排序与深度
3. **裁剪**：`RectMask` 走着色器，复杂遮罩需知性能成本
4. **事件**：实现对应 `I*Handler` 接口；确保场景有 `EventSystem`/`UIRaycaster` 路径
5. **编辑器扩展**：优先 CustomEditor / PropertyEditor，而不是改 Paper 内部
6. **热重载**：UI 组件同样依赖 `Identifier` 与 Dispose 断链规则
