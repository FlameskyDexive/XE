# 14 · 构建与发布

---

## 14.1 目标

将 Prowl 项目发布为 **独立桌面应用**：

- 编译用户程序集
- 收集并打包资源（松散或 `.prowlpak`）
- 生成玩家宿主（`DesktopPlayer : Game`）
- 按平台 RID 发布（Win / Linux / macOS）
- 可选自包含、裁剪、macOS App Bundle

## 14.3 BuildSettings

| 字段 | 说明 |
|------|------|
| `Scenes: List<SceneBuildEntry>` | 构建场景列表（需至少一个启用） |
| `Config` | Debug / Release |
| `OutputDirectory` | 输出目录（通常要求为空） |
| `PackagingMode` | 资源打包模式 |
| `AssetMode` | `AllAssets` / `DependenciesOnly` |
| 平台配置 | `PlatformBuildProfile` 列表 |

### DesktopBuildProfile 典型项

- RID：`win-x64` / `linux-x64` / `osx-x64` / `osx-arm64`
- `SelfContained`
- `PublishTrimmed`
- 窗口默认宽高

`AssetPackagingMode` 定义见 Runtime `AssetPackagingMode.cs`。

## 14.4 Desktop 构建流水线

`DesktopBuildPipeline` 主要步骤：

```
1. ScriptCompiler.CompileAll          # 用户程序集
2. 校验 ≥1 个启用场景；输出目录可用
3. 自动保存当前场景；重新导入构建场景
4. AssetCollector.Collect
     ├─ 从启用场景出发依赖遍历
     ├─ 生成 Resources 映射
     └─ 跳过 IsEditorOnlyAsset 导入器产物
5. 确保缓存就绪；复制/打包资源到 Content/
6. 生成玩家宿主项目 (DesktopPlayer : Game)
     └─ dotnet publish -r <RID> ...
7. 复制插件；整理输出
8. macOS：可选 .app Bundle 结构
9. 可选：构建后运行
```

进度通过 `BuildProgress` 报告；结果为 `BuildResult`。

## 14.5 资源收集策略

**类:** `AssetCollector`

| AssetMode | 行为 |
|-----------|------|
| `DependenciesOnly` | 仅打包场景与引用链上的资源（推荐发布） |
| `AllAssets` | 打包 Assets 下全部适用资源 |

依赖来源：

- 场景序列化中的 `AssetRef` / `AssetID`
- 递归资源字段
- Resources 特殊映射（若使用）

排除：

- 编辑器专用资产（脚本、asmdef、编辑器 DLL 等）
- 未引用且非 AllAssets 的内容

输出侧由 `PlayerAssetDatabase` 在运行时解析（松散文件或 ProwlPak + manifest）。

---

## 14.6 玩家宿主

构建生成的玩家程序：

- 继承 `Game`
- 启动时设置 `Application.IsPlaying = true`、`DataPath` 等
- 安装 `PlayerAssetDatabase` 为 `AssetDatabase.Current`
- 加载首个启用场景
- 进入 `Run(title, width, height)` 循环

**无 Editor 依赖**——只链接 Runtime 与用户 Game 程序集、插件与原生库。

## 14.7 CLI 无头构建

```text
Prowl.Editor --buildmode <projectPath> --output <outputPath>
```

适用于 CI。需本机安装 .NET 10 SDK 与对应平台原生依赖。

其他 CLI：

- `--project <path>` 打开项目
- `--restore-scene <path>` 热重载失败恢复场景

---

## 14.8 平台与原生库

Runtime 将 `Libraries/**` 下原生库复制到 `runtimes/` 布局：

| 目录示例 | 平台 |
|----------|------|
| `win-x64` / `win-x86` | Windows |
| `linux-x64` / `linux-arm` / `linux-arm64` | Linux |
| `osx-x64` / `osx-arm64` | macOS |
| `android-arm` / `android-arm64` | Android（库存在；桌面流水线以 Desktop 为主） |

MiniAudio、OpenGL 相关绑定随 Silk.NET / 原生包分发。发布时确保 RID 与库匹配。

---

## 14.9 裁剪与 AOT 注意

- Runtime 标注为 **AOT 友好** 方向，但反射（组件 `Activator.CreateInstance`、序列化、注册表扫描）需保留根
- `PublishTrimmed` 开启时注意：
  - 用户脚本被反射创建的类型
  - Echo 序列化字段
  - 着色器/资源路径字符串
- 建议先 **非裁剪** 验证，再逐步加裁剪与描述符

---

## 14.10 发布检查清单

1. 构建场景列表正确，首场景可进入
2. `AssetMode` 符合预期；关键资源无丢失
3. 输入绑定 / 音频 / 插件在目标平台验证
4. 窗口尺寸与全屏策略
5. 日志路径与 `DataPath` 可写
6. 许可证与第三方原生库合规（如 MiniAudio）
7. 在干净机器或 CI 产物上做一次冒烟运行
